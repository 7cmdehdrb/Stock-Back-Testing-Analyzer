# 프로젝트 전체 설명

## 프로젝트 목적

**Stock Back Testing Analyzer**는 사용자가 보유한 주식 포트폴리오 CSV를 업로드하면 과거 주가 데이터를 기반으로 포트폴리오 성과를 분석하는 Flask 웹 애플리케이션입니다. 단순 수익률뿐 아니라 벤치마크 대비 성과, 위험 조정 수익률, 자산 배분, 환율 반영 결과를 함께 보여 주어 투자 포트폴리오를 더 입체적으로 검토할 수 있도록 돕습니다.

> 이 프로젝트의 분석 결과는 투자 판단을 돕기 위한 참고 정보이며, 투자 수익을 보장하지 않습니다.

## 주요 사용자 흐름

1. 사용자가 회원가입 또는 로그인을 진행합니다.
2. 보유 종목 정보가 담긴 CSV 파일을 업로드합니다.
3. 분석 시작일, 벤치마크, 기준 통화, 현금 보유액 등을 선택합니다.
4. 애플리케이션이 Yahoo Finance에서 가격 데이터를 가져오고 로컬 SQLite 캐시에 저장합니다.
5. 포트폴리오 수익률, 벤치마크 비교, 위험 지표, 자산 배분 차트를 제공합니다.
6. 로그인 사용자는 분석한 포트폴리오를 저장하고 마이페이지에서 다시 확인할 수 있습니다.

## 핵심 기능

### 포트폴리오 업로드 및 분석

- CSV 기반 포트폴리오 입력
- 미국/한국 주식 티커 처리
- 동일 티커 중복 행 병합
- 보유량과 평균 단가를 기반으로 한 보유 현황 계산
- 시작일 기준 백테스트 수익률 계산

### 성과 지표 계산

- 누적 수익률
- 연평균 성장률(CAGR)
- 변동성
- 샤프 비율
- 소티노 비율
- 알파
- 베타
- 최대 낙폭(MDD) 성격의 손실 구간 분석

### 벤치마크 비교

- 사용자가 입력한 일반 티커 벤치마크와 비교할 수 있습니다.
- 블랙록, 버크셔 해서웨이 등 샘플 포트폴리오 CSV를 활용한 헤지펀드형 벤치마크도 지원합니다.

### 다중 통화 및 환율 처리

- USD와 KRW 기준 통화를 지원합니다.
- USD/KRW 환율 데이터를 가져와 포트폴리오 평가액과 수익률에 반영합니다.
- 환율 데이터 역시 로컬 캐시를 활용합니다.

### 사용자 계정 및 저장 기능

- 이메일 인증 기반 회원가입
- 비밀번호 해시 저장
- 로그인/로그아웃
- 닉네임 중복 확인 및 수정
- 사용자별 포트폴리오 저장/조회/삭제
- 랭킹 페이지 제공

### 관리자 기능

- 별도 관리자 로그인 페이지
- Flask-Admin 기반 데이터 관리 화면
- 주가 캐시, 저장된 포트폴리오, 사용자, 이메일 인증, 환율 캐시 관리

### AI 분석 기능

- OpenAI API 키가 설정된 경우 포트폴리오 분석 결과에 대한 AI 해설을 요청할 수 있습니다.
- 사용자별 요청 제한 및 메모리 캐싱 구조가 포함되어 있습니다.

## 데이터 흐름

```text
CSV 업로드
  ↓
CSV 파싱 및 필수 컬럼 검증
  ↓
티커 정규화 및 중복 병합
  ↓
Yahoo Finance 가격/환율 데이터 조회
  ↓
SQLite 캐시 저장 또는 재사용
  ↓
수익률·위험 지표·자산 배분 계산
  ↓
HTML 템플릿과 Chart.js 기반 결과 표시
  ↓
선택 시 사용자 포트폴리오 저장
```

## 데이터베이스 모델 개요

| 모델 | 역할 |
| --- | --- |
| `StockPriceCache` | 티커와 날짜별 종가 캐시 |
| `SavedPortfolio` | 사용자가 저장한 포트폴리오 CSV와 분석 조건 |
| `User` | 이메일, 닉네임, 계정 유형, 관리자 여부 등 사용자 정보 |
| `EmailVerification` | 이메일 인증 코드와 만료/인증 상태 |
| `ExchangeRateCache` | USD/KRW 환율 OHLC 데이터 캐시 |

| 변수 | 필수 여부 | 설명 |
| --- | --- | --- |
| `SECRET_KEY` | 권장/필수 | Flask 세션과 비밀번호 해시에 사용되는 비밀 키 |
| `OPENAI_API_KEY` | 선택 | AI 포트폴리오 분석 기능 사용 시 필요 |

| `SENDER_EMAIL` | 선택 | 발신 이메일 주소 |
| `SENDER_PASSWORD` | 선택 | 발신 이메일 비밀번호 또는 앱 비밀번호 |

## CSV 입력 형식

필수 컬럼은 다음과 같습니다.

| 컬럼 | 설명 | 예시 |
| --- | --- | --- |
| `티커` | Yahoo Finance에서 조회 가능한 티커 | `AAPL`, `005930.KS` |
| `보유량` | 보유 수량 | `10`, `50` |
| `국가` | 종목 국가 | `미국`, `한국` |

선택 컬럼 예시는 다음과 같습니다.

| 컬럼 | 설명 |
| --- | --- |
| `평단가` | 평균 매입 단가 |
| `종목명` | 표시용 종목명 |
| `통화` | `USD`, `KRW` 등 |
| `분류` | 우량주, ETF 등 사용자 정의 분류 |
| `섹터` | 기술, 금융 등 사용자 정의 섹터 |

## 실행 개요

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -c "from app import init_database; init_database()"
python app.py
```

기본 로컬 접속 주소는 `http://localhost:8000`입니다.

## 운영 시 주의 사항

- `stock_cache.db`는 로컬 SQLite 데이터베이스이므로 운영 배포 시 백업 정책이 필요합니다.
- `SECRET_KEY`, SMTP 비밀번호, OpenAI API 키는 저장소에 커밋하지 말고 환경 변수로 관리해야 합니다.
- Yahoo Finance 데이터는 외부 서비스 상태와 티커 지원 여부에 영향을 받습니다.
- 투자 분석 결과는 데이터 품질, 환율 기준, 시작일, 벤치마크 선택에 따라 달라질 수 있습니다.
